All about azure VMs (IaaS)
  • IaaS and virtual machines make it quick and economical to scale up dev-test environments up and down.
  • Running websites by using IaaS and virtual machines can be less expensive than traditional web hosting.
  • IaaS is useful for handling unpredictable demand and steadily growing storage needs.
  • In IaaS, You can simplify planning and management of backup and recovery systems.
  • Consider high-performance computing
  • big data analysis which IaaS economically provides.
  • extended datacenters - Add capacity to your datacenter by adding virtual machines in Azure
Plan VMS
  • Network, VM name, location, size, pricing models, Azure storage, OS
  • By default, services outside the virtual network can't connect to services within the virtual network.
  • You can, however, configure the network to allow access to the external service, including your on-premises servers.
  • Note that Network addresses and subnets aren't trivial to change after they're configured. think about topology before putting VMs
  • VM name upto 15 chars on Win VM and upto 64 on linux vms
  • Try to include these in the VM name
  • Environment or purpose, location, Instance, product or service, Role
  • The location of the machine can improve performance and ensure you meet any legal, compliance, or tax requirements.
  • Location can also be choosen that's close to your users/customers
  • There are price differences between locations
  • Determine VM size based on type of the workload that runs on VM.
  • Specify disk size and performance tier (standard or Premium)
  • Azure Managed Disks handle the azure storage account creation and management in the background for you
  • Compute expenses priced on a per hour basis but billed on per-minute basis
  • Consumption-based or REserved VM instances - Choose these based on business need
  • Reserved VM gives upto 72% price savings compared to pay-as-you-go pricing
  • Storage costs are charged seperately for azure storage used by the VMs
virtual machine sizes
  • General purpose - have a balanced CPU-to-memory ratio.
  • Compute optimized - have a high CPU-to-memory ratio.
  • Memory optimized - to have a high memory-to-CPU ratio.
  • Storage optimized - to have high disk throughput and I/O.
  • GPU - heavy graphics rendering and video editing.
  • High performance computes - the fastest and most powerful CPU virtual machines with optional high-throughput network interfaces (RDMA).
  • Azure allows you to change the virtual machine size when the existing size no longer meets your needs.
  • Caution - Resizing a machine might require a restart that can cause a temporary outage or change configuration settings such as the IP address
VM Storage(VHD)
  • All disks are stored as virtual hard disks (VHDs)
  • A VHD is like a physical disk in an on-premises server but, virtualized.
  • Every virtual machine has one attached operating system disk
  • The OS disk is registered as a SATA drive (Serial Advanced Technology Attachment) and labeled as the C drive by default.
  • On Win VMs, temporary disk is labeled as D drive by default, stores pagefile.sys file
  • OnVM, temp disk is typically /dev/sdb. This disk is formatted and mounted to /mnt by azure linux agent
  • Don't store data on temp disk that's business critical, it can lost during VM reboot or any config (IP) changes
Data Disks
  • A data disk is a managed disk that's attached to a virtual machine to store application data
  • choose Premium Storage to gain high-performance, low-latency disk support for your virtual machines with input/output (I/O)-intensive workloads (SSDs)
  • you can attach several Premium Storage disks to a virtual machine up to 256 TB of storage per virtual machine
  • Premium storage can achieve 80,000 I/O operations per second (IOPS) per virtual machine and 2k MB/S/vm
  • Azure-managed disks are stored as page blobs, which are a random IO storage object in Azure.
Important
  • Use Azure Bastion to connect to VMs, Bastion connects privately and VM need not have a public address

Control Access to Az Storage_SAS & more
  • A shared access signature (SAS) provides secure, delegated access to resources in your storage account.
  • Access Azure Storage ==> Public Access, Az AD, Shared key, Shared Access Signature(SAS)
  • Public access is also known as anonymous public read access for containers and blobs
  • 1. Configure storage a/c to allow public access by setting AllowBlobPublicAccess
  • 2. You can enable anonymous access only if anonymous access has been allowed for storage account
  • AZ AD ==> Use the Azure AD option to securely access Azure Storage without storing any credentials in your code.
  • First, you authenticate a security principal that returns an OAuth 2.0 token if successful
  • This token is then passed to Azure Storage to enable authorization to the requested resource.
  • Shared key ==> Azure Storage creates two 512-bit access keys for every storage account that's created
  • You share these keys to grant clients access to the storage account. These keys grant anyone with access the equivalent of root access to your storage.
  • recommendation is to manage storage keys with Azure Key Vault
  • SAS ==> lets you grant granular access to files in Azure Storage
  • 1. User delegation SAS
  • 2. ervice SAS
  • 3. Account SAS
  • With SAS, you control what they access.
What are stored access policies?
  • Short one!- SAS is specific to an image and has expiry. This can be solved by SAS with stored access policies
  • More! Create SAS tokens and associate them with stored access policies
Upload, download and manage data with Azure Storage Explorer
  • Pre-Azure Data Lake Storage is a secure cloud platform that provides scalable, cost-effective storage for big data analytics
  • based on Apache Hadoop. can store unstructured and structured data
  • Storage Explorer is available on Windows, macOS, and Linux.
  • Storage Explorer supports two emulators ==> Azure Storage Emulator and Azurite.
  • Azure Storage Emulator uses a local instance of Microsoft SQL Server 2012 Express LocalDB.
  • Azurite, which is based on Node.js, is an open-source emulator that supports most Azure Storage commands through an API

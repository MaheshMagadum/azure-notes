VMs
  • Total control over the operating system (OS).
  • The ability to run custom software
  • To use custom hosting configurations.
  • IaaS offering - you still need to configure, update, and maintain the software that runs on the VM.
Scale VMs
  • Can group VMs together to provide high availability, scalability and redundancy.
  • Azure can manage grouping of VMS for you with features such as scale sets and availability sets (check this!)
VM Scale Sets
  • VM scale sets let you create and manage a group of identical, load-balanced VMs
  • You would also have to monitor the utilization to determine if you need to increase or decrease the number of VMs.
  • Instead with VM Scale Sets, Azure automates most that work. Scale sets allow ou to centrally manage, configure & update a large no of VMs in minutes.
  • The number of VM instances can automatically increase or decrease in response to demand, or you can set it to scale based on a defined schedule
  • Virtual machine scale sets also automatically deploy a load balancer to make sure that your resources are being used efficiently
  • Ex:- you can build large-scale services for areas such as compute, big data, and container workloads.
VM Availability Sets
  • helps build a more resilient, highly available environment
  • Availability Sets do this by below
  • Update domain:- Groups VMs that can be rebooted at the same time. Imp-only one update domain grouping will be offline at a time.
  • update process is given a 30 minutes time to recover time to recover before maintenance on next update domain starts
  • Fault domain:- Groups VMs by common power source and network switch.
  • By default, an availability set splits VMs across up to three fault domains.
  • his helps protect against a physical power or networking failure by having VMs in different fault domains
  • thus being connected to different power and networking resources
  • Best of all! there’s no additional cost for configuring an availability set. You only pay for the VM instances you create
When to use VMs
  • During testing and development
  • When running applications in the cloud
  • When extending your datacenter to the cloud. Applications like SharePoint can then run on an Azure VM instead of running locally
  • During disaster recovery - you can get significant cost savings by using an IaaS-based approach to disaster recovery
  • Move to the cloud with VMs - you’re responsible for maintaining the installed OS and software
VM Resources
  • Flexibility to choose VM resources such as OS image, Disk, etc

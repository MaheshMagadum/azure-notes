What is a storage account?
  • Azure data services, such as Azure SQL and Azure Cosmos DB, are managed as independent Azure resources and cannot be included in a storage account
  • Separating proprietary data and public data will also require separate storage accounts.
  • Regions/locations requirements also need seperate storage accounts
  • In general, increased diversity means an increased number of storage accounts.
  • You could place your critical data into a storage account with geo-redundant storage AND
  • You could put your non-critical data in a different storage account with locally redundant storage.
  • The core advice here is to choose the Resource Manager deployment model and the StorageV2 (general purpose v2) account kind for all your storage accounts

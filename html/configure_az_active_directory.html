Things to know about Azure Active Directory benefits and features
  • Azure AD is Microsoft's multi-tenant cloud-based directory and identity mgmt service.
  • Azure AD helps to support user access to resources and applications such as
  • Internal resources and apps located on your coporate network.
  • External resources like Microsoft 365, the AZ portal and SaaS applications.
  • Cloud apps developed for the organization. Consider below points for ex
  • 1. Windows Server Active Directory is using Kerberos and NTLM authentication to on-pre apps
  • 2. AZ active directory uses SAML, Oauth, Open ID, Ws-Federation to Azure Cloud resources.
Azure AD Features
  • SSO - AZ AD provides SSO to web apps on cloud and on-pre. users can sign in with same set of creds to access all their apps
  • Ubiquitous device support - AZ AD works with iOS, macOS, Android and Win devices and offers a common experience across all devices.
  • Secure remote access - AZ AD enables secure remote access for on-pre web apps. This can include multifactor auth (MFA), conditional access policies and group-based access mgmt
  • Cloud extensibility - AZ AD can extend to cloud to help manage a consistent set of users, grps, pwds & devices across environments
  • Sensitive data protection - AZ AD offers unique identity protection capabilties to secure sensitive data and apps.
  • self-service support - AZ AD lets you delegate tasks to company emps that might otherwise be completed by admins with higher access privileges.
  • And password management through verfication steps can reduce helpdesk calls and enhance security.
Things to consider when using Azure AD features
  • Consider enabling SSO - Enable SSO access for users to connect to the cloud or use on-prem apps
  • And AZ AD SSO supports MS 365 and many SaaS apps such as Salesforce, Workday, DocuSign, ServiceNow and Box
  • Consider UX and device support - Consistent user experience that works for all devices and directory access points
  • And you can design custom company portals and personalized web-based access for employees that lets them connect with their existing work creds
  • Consider benefits of secure remote access - protect your on-pre web apps by using MFA and access policies
  • Consider advantages of cloud extensibility - connect Active Directory and other on-prem dirs in cloud to Azure AD in just few steps
  • And you can make it easier for your admins to manage sae users, groups, pwds and devices across all supported envs
  • Consider advanced protection for sensitive data - Enahance sec of sensitive data by built-in protection features of AZ AD
  • Consider reduced costs, self-service options.
Describe Azure Active Directory concepts
  • Identity ==> is an object that can be authenticated. it can be user with pwd. It can also be apps, servers that require auth by secret keys or certs
  • Account ==> is an identity that has data associated with it. To have account, you must first have a valid identity.
  • AZ AD account ==> AZ AD account is an identity that's created via Azure AD or another MS cloud service (MS 365) for ex.
  • Azure AD account is also called a work or school account.
Azure tentant(directory)
  • AZ tenant is signle dedicated and trusted instance of Azure AD. Each tenant (also called directory) represents a single organixation.
  • When an org signs up for MS cloud service subscription, a new tenant is auto created.
  • You can create multiple tenants of instances
Azure subscription
  • is used to pay for Azure and cloud services. Each subscription is joined to a single tenant.
Active Directory Domain Services
  • AD DS is the traditional deployment of Windows Server-based Active Directory on a physical or virtual server
  • AD DS also includes Active Directory Certificate Services (AD CS), Active Directory Lightweight Directory Services (AD LS), Active Directory Federation Services (AD FS), and Active Directory Rights Management Services (AD RMS)
  • IaaS workloads depend specifically on AD DS, Azure Active Directory is recommended to be used
Things to consider when using Azure AD rather than AD DS
  • consider the following characteristics that distinguish Azure AD from AD DS.
  • Identity solution ==> AD DS is primarily a directory service, while Azure AD is a full identity solution
  • Azure AD is designed for internet-based applications that use HTTP and HTTPS communications
  • Communication protocols ==> Because Azure AD is based on HTTP and HTTPS, it doesn't use Kerberos authentication.
  • And SAML, WS-Federation, and OpenID Connect for authentication are HTTP/S protocol implementation
  • Federation services ==> Azure AD includes federation services, and many third-party services like Facebook.
  • Flat structure ==> Azure AD users and groups are created in a flat structure. There are no organizational units (OUs) or group policy objects (GPOs).
  • Managed service ==> Azure AD is a managed service. You manage only users, groups, and policies.]
  • If you deploy AD DS with virtual machines by using Azure, you manage many other tasks, including deployment, configuration, virtual machines, patching, and other backend processes.

Implement Azure Active Directory join
  • The Azure AD join feature works with SSO to provide access to organizational apps and resources, and to simplify Windows deployments of work-owned devices.
Things to know about the Azure AD join feature
  • SSO ==> Joined devices offer SSO access to your Azure-managed SaaS apps and services
  • Enterprise state roaming ==> Starting in Windows 10, your users can securely synchronize their user settings and app settings data to joined devices.
  • Access to Microsoft Store for Business ==> When your users access Microsoft Store for Business by using an Azure AD account, they can choose from an inventory of applications pre-selected by your organization.
  • Windows Hello ==> Provide your users with secure and convenient access to work resources from joined devices.
  • Restriction of access ==> Restrict user access to apps from only joined devices that meet your compliance policies.
  • Seamless access to on-premises resources ==> Joined devices have seamless access to on-premises resources, when the device has line of sight to the on-premises domain controller.
Things to consider when using joined devices
  • Connect your device to Azure AD in one of two ways
  • Register ==> your device to Azure AD so you can manage the device identity.
  • Join ==> your device, which is an extension of registering a device.
  • Combine registration with a mobile device management (MDM) solution like Microsoft Intune, to provide other device attributes in Azure AD
  • You can create conditional access rules that enforce access from devices to meet organization standards for security and compliance.
  • Although AD Join is intended for organizations that don't have an on-premises Windows Server Active Directory infrastructure, it can be used for other scenarios like branch offices.
Azure Active Directory self-service password reset
  • The Azure Active Directory self-service password reset (SSPR) feature lets you give users the ability to bypass helpdesk and reset their own passwords.
Things to know about the Azure AD SSPR feature
  • SSPR requires an Azure AD account with Global Administrator privileges to manage SSPR options
  • This account can always reset their own passwords, no matter what options are configured.
  • SSPR uses a security group to limit the users who have SSPR privileges.
  • All user accounts in your organization must have a valid license to use SSPR.
Things to consider when using SSPR
  • Consider who can reset their passwords
  • Consider your authentication methods. Determine how many authentication methods are required to reset a password, and select the authentication options for users.
  • Your system must require at least one authentication method to reset a password.
  • A strong SSPR plan offers multiple authentication methods for the user. Options include email notification, text message, or a security code sent to the user's mobile or office phone
  • You can require security questions to be registered for the users in your Azure AD tenant.
  • You can configure how many correctly answered security questions are required for a successful password reset.
  • Consider combining methods for stronger security.

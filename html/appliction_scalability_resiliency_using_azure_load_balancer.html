Azure load balancer
  • Use Load Balancer to scale applications and create high availability for your virtual machines and services
  • Load balancers use a hash-based distribution algorithm. By default, a five-tuple hash is used to map traffic to available servers
  • 5-tuple hash => source IP, Source Port, Destination IP, Destination Port, Protocol (TCP or UDP)
  • SLA=>99.95%=>Protection from hardware failures within datacenters
  • SLA 99.99%=>Protection from entire datacenter failure
  • basic load balancers and standard load balancers.
  • BAsic allow => Port forwarding, Automatic reconfiguration, Health probes, outbound connections through SNAT (source network address translation), Diagnostics thru azure log analytics for public-facing load balancers
  • Basic load balancers can be used only with a single availability set or scale set.
  • Standard load balancers support all of the basic load balancer features.
  • Standard allows=>HTTPS health probes, availability zones, diagnostics thru Azure monitor, HA ports, outbound rules, a guaranteed SLA 99.99%
  • External => An external load balancer operates by distributing client traffic across multiple virtual machines. An external load balancer permits traffic from the internet.
  • Internal => An internal load balancer distributes a load from internal Azure resources to other Azure resources
Check
  • Which configuration is required to configure an internal load balancer? Virtual machines should be in the same virtual network
  • Which one of the following statements about external load balancers is correct? They have a public IP address.
Notes
  • 5-tuple has or Source IP affinity (session affinity or client IP affinity)
  • To map traffic to the available servers, the source IP affinity mode uses a two-tuple hash (from the source IP address and destination IP address) or a three-tuple hash (from the source IP address, destination IP address, and protocol type).
  • The default five-tuple hash in Load Balancer is incompatible with RDP service, use use source IP affinity instead.
